# M050: Overnight Run Report

After a long run, users need a concise report of what happened overnight: tasks attempted, stops, evidence, guardrails, and recommended next action. The report should be generated from stored planning state and recent run activity.

## Success Criteria

- EXECUTE/SHIP can produce an overnight run report.
- The report includes attempts, stops, evidence, guardrails, and next work.
- The report survives restart because it is derived from database state.
