# M029: Conflict And Scope Guardrails

Autonomous execution needs explicit stop conditions when scope changes, files conflict, tests fail, or the task no longer matches the accepted plan. These guardrails should be visible and project-specific.

## Success Criteria

- Guardrails are represented as durable planning policy.
- The UI makes stop conditions visible before starting queued work.
- Projections include stop conditions for handoff sessions.
