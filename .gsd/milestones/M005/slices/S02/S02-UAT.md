# S02 UAT: Requirements Contract Review

## Scenario

1. Create a Plan Builder plan.
2. Answer the REQUIREMENTS questions.
3. At the REQUIREMENTS depth gate, review the requirements contract.
4. Save the requirements contract.
5. Continue through PLAN and accept the generated plan.
6. Restart the desktop app and reopen Plan Builder.

## Expected Result

- The contract shows four saved requirement rows.
- Each row includes id, title, class, status, source, owner, and validation state.
- The saved rows remain visible after restart.
- `.gsd/REQUIREMENTS.md` contains the same requirement rows and traceability table.
