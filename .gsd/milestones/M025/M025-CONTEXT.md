# M025: Execution Handoff Context

## Vision

When a user starts execution, each task should open with the planning context needed to do the work: accepted task details, linked requirements, relevant research, phase model preference, and verification expectations.

## Success Criteria

- Task sessions can be launched or opened with a generated execution brief.
- Execution briefs include task acceptance, dependencies, requirement coverage, and relevant planning evidence.
- Phase model preference is available to the execution handoff.
- Execution evidence can be linked back to the task queue.
- Restart coverage proves task-session links and evidence survive.

## Non-Goals

- Fully autonomous execution without user review.
- Replacing normal chat/session surfaces.
- Running external provider calls in deterministic core tests.
- Changing the accepted PLAN schema outside the needed handoff fields.
