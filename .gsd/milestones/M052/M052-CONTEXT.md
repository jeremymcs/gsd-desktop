# M052: GSD Source Proof Visual Language

The Plan Builder needs a stronger visual identity drawn from GSD itself: database-backed planning, generated projections, workflow gates, evidence, and next-work state. The reference direction is a dark desktop workbench with source-code proof panels and an operational inspector rail.

## Success Criteria

- The no-plan state shows GSD source/projection proof instead of generic setup chrome.
- The right rail reinforces GSD terminology and durable artifacts.
- The design remains app-native, dense, and consistent with the existing Plan Builder surface.
